# TMUX Agents Manager

Имитация tmux сессии для Windows с управлением различными агентами и сервисами.

## Описание

Этот проект предоставляет альтернативу tmux для Windows, позволяя управлять несколькими агентами в отдельных "окнах" (процессах). Система поддерживает:

- **Telegram боты** (aiogram)
- **Планировщик задач** (apscheduler)
- **Мониторинг и HTTP сервисы** (aiohttp)
- **Автоматизация браузера** (playwright)
- **MCP серверы** (context7, puppeteer, brave-search, memory)

## Установка

1. Убедитесь, что Python 3.7+ установлен
2. Скачайте файлы:
   - `tmux_agents.py` - основной скрипт
   - `agents_config.json` - конфигурация

## Использование

### Запуск сессии агентов

```bash
py tmux_agents.py
```

### Доступные команды

- `list` - показать все активные окна
- `kill <name>` - остановить конкретное окно
- `restart <name>` - перезапустить окно
- `exit` - выйти и остановить все процессы

### Примеры окон

```
agents> list

Активные окна в сессии 'agents':
  telegram-bots: running (PID: 12345)
  scheduler: running (PID: 12346)
  monitoring: running (PID: 12347)
  browser-automation: running (PID: 12348)
  mcp-servers: running (PID: 12349)
```

## Настройка

### Изменение команд окон

Отредактируйте `agents_config.json`:

```json
{
  "windows": {
    "telegram-bots": {
      "command": "python your_telegram_bot.py",
      "working_directory": "./bots",
      "auto_restart": true
    }
  }
}
```

### Добавление новых окон

Добавьте новую запись в конфигурацию:

```json
"new-agent": {
  "description": "Новый агент",
  "command": "python new_agent.py",
  "working_directory": "./agents",
  "auto_restart": true
}
```

## Особенности

- **Кроссплатформенность**: Работает на Windows, Linux, macOS
- **Автоматический перезапуск**: Опция для автоматического перезапуска окон
- **Управление процессами**: Полный контроль над запущенными процессами
- **Логирование**: Сохранение логов работы системы
- **Конфигурируемость**: JSON-based конфигурация

## Безопасность

- Процессы запускаются в отдельных подпроцессах
- Корректное завершение всех процессов при выходе
- Обработка сигналов прерывания

## Разработка

Для добавления новых функций:

1. Отредактируйте `AgentsManager` класс в `tmux_agents.py`
2. Обновите конфигурацию в `agents_config.json`
3. Протестируйте изменения

## Troubleshooting

### Проблемы с запуском

- Убедитесь, что Python доступен в PATH
- Проверьте права доступа к рабочим директориям
- Проверьте корректность команд в конфигурации

### Процессы не завершаются

- Используйте `kill <name>` для принудительной остановки
- Проверьте логи на ошибки в процессах
